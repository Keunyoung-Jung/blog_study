프론트엔드와 백엔드의 기본 구성

  • 클라이언트 (Request)
  • 웹 서버 (Response)
    • 백엔드
    • 프론트엔드
    • 데이터 베이스

웹서비스 개발과 파이썬 flask를 이용한 개발

  • 1세대 : USER → INTERNET → static HTML이용
  • 2세대 : USER → INTERNET → Create request-based HTML CGI+DB
  • 3세대 : USER → INTERNET → MVC 패턴 기반 프레임워크 활용
    • MVC : Model-View-Control 패턴으로 구조화된 프레임워크 사용
  • 4세대 : OpenAPI, RestAPI 를 혼합하여 다양한 서비스 제공, 다양한 웹서비스 개발
    • 기능별 백엔드/프론트엔드를 구현해서 분산처리를 통해 빠른 서비스 제공

웹 백엔드/프론트엔드에 프레임워크를 사용하는 이유

  • 변수/조건/반복 으로 만들기에는 굉장한 시간 필요
  • 일정 부분 프레임워크의 장점 활용 필요 (단, 너무 복잡한 프레임워크는 단점 고려가 필요)

풀스택 프레임워크와 마이크로 프레임워크

  • 풀스택 프레임워크 : JAVA Spring, Python Django, Ruby on Rails
    • 요청/응답 수상화, 세션상태 관리, 사용자 인증/권한 관리, 웹페이지 템플릿, URL매핑, 데이터베이스 접근, 보안, 캐시, 데이터 접근 추상화 등 다양한 기능 제공
    • 다양한 프레임워크 기능을 이해하고, 불필요한 부분까지 구축해야되는 경우가 있음
  • 마이크로 프레임워크 : 최소 기능만 제공, 확장을 통해 사용해야한다.

URL이란

  • Uniform Resource Locator
  • 인터넷 상의 자원 위치 표기를 위한 규약

REST

  • REST(REpresentational State Transfer)
    • 자원(resource)의 표현(representation)에 의한 상태 전달
    • HTTP URI를 통해 자원을 명시하고, HTTP Method를 통해 자원에 대한 CRUD Operation 적용
      • CRUD Operation와 HTTP Method
        • Create : 생성 (POST)
        • Read : 조회 (GET)
        • Update : 수정 (PUT)
        • Delete : 삭제 (DELETE)

REST API

  • REST 기반으로 서비스 API를 구현한 것
  • 마이크로 서비스, OpenAPI 등에서 많이 사용됨
  • httpie로 REST API 확인
#간단하게 REST 보내서 확인
http GET http://localhost:8080/test

CDN이란?

  • CDN(Contents Delivery Network) 은 지리적, 물리적으로 떨어져 있는 사용자에게 컨텐츠 제공자의 컨텐츠를 더 빠르게 제공할 수 있는 기술을 의미
  • 사용자가 멀리 있는 서버로부터 컨텐츠를 다운로드 받으면, 시간이 오래걸리므로, 사용자와 가까운 곳에 위치한 Cache Server에 해당 컨텐츠를 저장해놓고, 컨텐츠 요청시, 서버가 아닌 Cache Server가 응답을 주는 기술

로깅(logging)다루기

  • 서버는 24시간 동작하므로, 문제가 있을 때, 어떤 문제가 있었는지, 파악하기 위해 로깅 기능을 사용함
  • 사용자 모니터링, 해킹 확인등 다양한 상용화시 문제에 대해서도 로깅 기능을 활용할 수 있음

간단한 logging 라이브러리 사용법

  • 파이썬에는 로그를 다루는 logging 라이브러리가 있음
  • 로딩 정보는 레벨이 있음
  • 로깅 정보는 로그의 레벨에 따라서 출력을 제한 할 수 있음
    • DEBUG > INFO > WARNING > ERROR > Critical
import logging
# 파일로 남기기 위해서는 filename='test.log' 파라미터, 어느 로그까지 남길 것인지를 level 로 설정 가능
logging.basicConfig(filename='test.log', level=logging.ERROR)

# 로그를 남길 부분에 다음과 같이 로그 레벨에 맞추어 출력해주면 해당 내용이 파일에 들어감
logging.debug("debug")
logging.info("info")
logging.warning("warning")
logging.error("error")
logging.critical("critical")

MVP (Minimum Viable Product)

  • 최소한의 기능을 구현한 제품
  • 고객의 피드백을 받아서, 기능을 점차적으로 개선
  • 최근 스타트업이 개발하는 방식

SQLAlchemy

  • Django 등에서 다양한 데이터베이스를 객체로 만들어서, 사용법을 통일하는 기법으로 SQLAlchemy 라는 기능을 사용함
  • SQL 이외에 추가적인 SQLAlchemy 기반 CRUD 기법을 익혀야 함
  • SQL, mongodb CRUD 에 익숙해지는 것이 보다 중요함
